📋 Dastur haqida

🎯 Fitness zallari uchun soddalashtirilgan CRM. Abonentlar kirish-chiqish harakatlari Hikvision FaceID terminallari orqali amalga oshiriladi. Mikroservis (ASP.NET) ma’lumotlarni events va actions jadvallariga yozadi.

✅ Har bir abonent 3 yo‘nalishda qarzdor bo‘lishi mumkin: Market Murabbiy Zal obunasi. To‘lovlar tizimi virtual (admin tomonidan son sifatida kiritiladi), haqiqiy pul emas, faqat obuna to‘lovlari Click orqali amalga oshiriladi (NodeJS mikroservis).

🏢 Tashkilotlar (zallar) uchun yagona obuna narxi superadmin tomonidan belgilanadi. Ikki rol: super_admin (platforma boshqaruvi) va director (zal egasi).

📌 Eslatma: Barcha interfeyslar O‘zbek (lotin/krill) + Rus tillarida, tungi/kunduzgi rejim qo‘llab-quvvatlanadi.

⚙️ Umumiy texnik talablar

  • ✅ Docker container + docker-compose (backend + frontend + postgresql)
  • ✅ PostgreSQL, tashqaridan ulanish uchun port ochilgan
  • ✅ Backend API (Nodejs yoki siz tanlagan til) JWT autentifikatsiya
  • ✅ Swagger login-parol bilan himoyalangan (.env)
  • ✅ RateLimiter: IP + tokenga asoslangan, har bir endpointga individual limit
  • ✅ 3 til: O‘zbek (lotin/krill) + Rus – JSON fayllar bilan
  • 🎨 Frontend: Tailwind CSS, moslashuvchan (mobil + desktop), dark/light mode
  • 📁 Yuklangan rasmlar endpoint /Upload orqali, token tekshiruvi bilan
  • 🗄️ Jadvallar: users, organizations, provinces, regions, tariffs, api_settings, monitors, news, feedbacks, messages, terminals, tasks, dates, graphics, person, payments, events, market, trainers, categories
  • 🔄 Person (abonent) o‘zgarishlari FaceID terminallariga task orqali sinxronlanadi (task_type 1..8)

🔐 Autentifikatsiya va xavfsizlik

🔄 JWT + Refresh token

Access token — 30 daqiqa, Refresh token — 3 kun. Refresh token revoke qilish mexanizmi refresh_tokens.sql orqali. Token expired bo‘lsa, avtomatik yangilanadi.

📌 Login jarayoni: users va organizations jadvallarida login/parol tekshiruvi, agar active=false bo‘lsa "akkaunt faollashtirilmagan" xabari, parol noto‘g‘ri bo‘lsa password_hint qaytariladi.

📲 Parolni tiklash (SMS)

Login yuboriladi -> recovery_requests jadvaliga yozuv, SMS xizmati (SendSMS metod bo‘sh, keyin to‘ldiriladi). 2 daqiqa amal qiladi, 5 urinish RateLimit orqali cheklangan.

recovery_requests: request_id, user_login, code, expired_at, revoked
🔒 Role-based: super_admin (users jadvali) va director (organizations). Login bo‘yicha tegishli rolni aniqlab, tegishli menyular yuklanadi.

👑 SuperAdmin - To‘liq boshqaruv paneli

Viloyatlar, tumanlar, zallar, tariflar, serverlar, yangiliklar, murojaatlar va boshqalar

📊 Dashboard (SuperAdmin)

✅ Faol zallar soni
⏳ Muddat o‘tgan zallar
💰 Oylik tushum (obuna to‘lovlari)
📈 Jami abonentlar

📍 Viloyatlar (CRUD)

  • Modal orqali qo‘shish/tahrirlash, o‘chirish (tuman mavjud bo‘lsa bloklanadi)
  • Excel download, detail page
  • Jadval pagination shart emas

🏙️ Tumanlar (Regions)

  • Province filter, qidiruv (3 belgi), pagination, detailda pastda “zal biriktirish” (region_id=0 bo‘lgan zallar chiqadi)
  • O‘chirishda barcha organizationlarning region_id=0 bo‘ladi
  • CRUD + Excel export

🏢 Zallar (Organizations)

  • Filter: viloyat, tuman, aktivlik; qidiruv; qo‘shilganda avtomatik api_settings va 2 ta default grafik yaratiladi
  • Jadvalda terminallar (2/3 online ko‘rinish), abonentlar soni
  • Detailda: aktiv switch, parol o‘zgartirish, tarif tanlash. Pastda region/monitor ma’lumoti

📦 Tariflar, ⚙️ Sozlamalar (api_settings)

Tariflar – oddiy CRUD. Sozlamalar – zal tanlab, opening_time, closing_time, price_per_user, max_users_count va hk. Superadmin barcha sozlamalarni o‘zgartira oladi.

🖥️ Serverlar (Monitors)

  • CRUD, monitor o‘chirilsa organization.monitor_id=0 bo‘ladi
  • Detailda monitorga zal biriktirish (organizations.monitor_id=0 bo‘lganlar)
  • Parol o‘zgartirish, aktiv/inaktiv qilish

📰 Yangiliklar & 💬 Murojaatlar & ✉️ Xabarlar

  • Yangilik: start tugmasi bosilganda barcha organizationlar uchun news_historyga yoziladi, update faqat startdan oldin mumkin
  • Murojaatlar: markup, is_seen, registration_letter filtrlari, barchasini o‘qildi belgilash
  • Xabarlar: zalga maxsus, admin tomonidan yoziladi, avtomatik yoki admin turi filteri
🔄 Profil (users) – o‘z profilini tahrirlash, parolni almashtirish, logout esa chap pastki burchakda.

🏋️ Director roli – Zal boshqaruvi

Hisobotlar, terminallar, market, ro‘yxatlar (abonentlar, murabbiylar, grafiklar) va sozlamalar

🖥️ Terminallar → Qurilmalar

  • CRUD, max_terminals_count tekshiruvi, o‘chirishda tegishli tasklar soft delete
  • Detailda terminal topshiriqlari (tasks) pagination bilan
  • Reset tugmasi – barcha tasklarni deleted=true va yangi task_type=5 yaratadi

📋 Terminallar → Topshiriqlar

  • Faqat ko‘rish, filter: terminal, status (0-kutilmoqda,1-bajarildi,2-xatolik)
  • Pagination, detailda terminal haqida ma’lumot

📅 Ish kunlari (Dates)

Yil/oy tanlanadi, switch orqali ta’til belgilanadi (dates.holiday). Jadvalda sanalar o‘zgaradi.

📈 Grafiklar (Graphics)

CRUD, dars jadvallari / majburiy kunlar. Download, pagination shart emas.

⚙️ Sozlamalar (Zal sozlamalari)

Faqat ruxsat berilgan maydonlar: opening_time, closing_time, price_per_user. Info icon orqali tushuntirishlar.

👤 Profil (organizations) – o‘z ma’lumotlarini tahrirlash, parolni o‘zgartirish. Tepadagi menyuda: yangiliklar soni (o‘qilmagan), xabarlar soni, murojaat yuborish tugmasi.

👥 Abonentlar (Person)

  • 📍 Asosiy jadval: avatar, ism, telefon, total_debt, zal qarzi (zal_debt), subscribe_end_date, access_count.
  • ⚡ Qarzlarni so‘ndirish: to‘lov modal orqali (qarz to‘liq yoki qisman so‘ndiriladi). Raschyot qilish – barcha qarzlarni is_important=false qiladi.
  • 🔄 Obuna uzaytirish: 1 oy + price_per_user bo‘yicha to‘lov, payment yoziladi.
  • 🎨 FaceID sinxronizatsiya: qo‘shish/update/delete/rasm o‘zgarishi -> har bir terminal uchun task (task_type 1,2,3,4,8) yaratiladi.
  • 🔍 Filterlar: active, muddat o‘tgan/tugagan, murabbiy bo‘yicha. Qidiruv 3 belgidan keyin.
  • 📄 Detailda: Market va trainer qarzlarini alohida so‘ndirish, to‘lovlar history (10 ta), harakatlar history (10 ta). Murabbiy biriktirish/uzaytirish.

🏆 Murabbiylar (Trainers)

  • CRUD, card ko‘rinishida, rasm, yutuqlar, narx, shogirdlar soni.
  • Detailda shogirdlar ro‘yxati, shogird qo‘shish, shogirdni o‘chirish – trainer_debt nollanadi va paymentlar is_important=false.
  • Murabbiy oylik tushum statistikasi (trainers jadvalida saqlanadi).
  • Faqat person.is_client=true shogird bo‘lishi mumkin.

🛒 Market (Online savdo)

  • Kategoriyalar (superadmin tomonidan boshqariladi), mahsulotlar CRUD, rasm yuklash.
  • Card ko‘rinishida, savat (localstorage) – mahsulot, miqdor, narx. Person tanlab, to‘lov yaratiladi (chiqim + kirim). Market payments category bo‘yicha.
  • Sotishda total_price va to‘langan summa farqi qarz sifatida qolishi mumkin.

📌 Kategoriyalar (Categories)

Superadmin tomonidan CRUD, icon yuklash, o‘chirilganda mahsulotlar ham o‘chadi.

📈 Hisobotlar modullari (Director)

📊 Dashboard (Zal uchun)

💰 Kunlik/haftalik/oylik tushum
⚠️ Muddat tugagan / kirish soni tugagan / 1 hafta qolganlar
🏃 Bugungi tashriflar, hozir zaldagilar, 1 hafta tashrif qilmaganlar

🚪 Harakatlar (Kirish/Chiqish events)

  • Eventlar FaceID dan keladi, qo‘lda qo‘shish mumkin (terminalId=0).
  • Filter: BeginDate/EndDate, person bo‘yicha qidiruv, is_client filtri.
  • Pagination, detailda person va terminal ma’lumotlari.

💸 To‘lovlar (Payments)

  • Kategoriya (Zal, Murabbiy, Market), kirim/chiqim radiobutton. Qo‘lda qo‘shish / o‘chirish (o‘chirilganda person balansi qayta hisoblanadi).
  • Filter: person, kategoriya, is_important (aktiv). Raschyot qilish tugmasi – tanlangan paymentlarni is_important=false qiladi.
  • Excel export, pagination. Person.is_client=false to‘lov yaratish taqiqlangan.

🤖 FaceID terminallar bilan sinxronizatsiya mexanizmi

📌 Task turlari (tasks jadvali):

  • type 1 – yangi person qo‘shish
  • type 2 – person ma’lumotlarini yangilash
  • type 3 – personni o‘chirish
  • type 4 – rasm yuklash (url)
  • type 5 – terminalni tozalash (reset)
  • type 8 – rasmni tozalash

⚙️ Person qo‘shilganda barcha terminallarga task yoziladi. Terminal har 10 sekundda tasklarni o‘qib sinxronlaydi.

✅ Person o‘chirilganda payments, events, actions o‘chiriladi va terminallarga delete task (type 3) yuboriladi.
